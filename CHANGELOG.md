All notable changes to Aether Suite are documented here.

## [2.3.2] – 2026-04-14  *(current)*

### 🔴 Critical Bug Fixes

#### [Rust + iOS] FD double-close (Use-After-Free)
**Before:** `AetherManager.swift` opened `FileHandle(forWritingTo:)` then called
`fileHandle.close()` after Rust had already closed the underlying fd.  
**After:** `AetherManager` uses `Darwin.open()` directly to obtain a raw fd.
Swift never wraps it in a `FileHandle`, so there is no second close.
Rust is the sole fd owner.

#### [Rust] `block_in_place` could panic on `current_thread` runtime
**Before:** `apply_patch` called `rt.block_on(async { block_in_place(|| …) })`.
`block_in_place` panics when invoked outside a multi-thread runtime context.  
**After:** CPU-bound patch work is dispatched via `std::thread::spawn().join()`,
which is safe regardless of the Tokio runtime flavor.

#### [Rust / Android / iOS] Hardcoded `"MY_SELF_UUID"` peer identity
**Before:** All outbound HTTP requests sent `?pid=MY_SELF_UUID`, making peer
identity lookup always fail (server couldn't find the key for `"MY_SELF_UUID"`).  
**After:** `AetherEngine::set_self_peer_id()` is exposed. Android sets it to
`ANDROID_ID`; iOS sets it to `identifierForVendor`. A random UUID is generated
as default if neither platform sets it.

#### [Rust] No integrity check after download
**Before:** Downloaded bytes were written to disk with no verification.  
**After:** SHA-256 is computed inline during streaming transfer. Mismatch returns
`AetherError::ChecksumMismatch` before the caller sees the file.

### 🟡 Moderate Fixes

#### [Rust] Rate limiting on download endpoint
**Before:** No limit on concurrent `/download` requests; a hostile peer could
exhaust Tokio worker threads.  
**After:** `Arc<Semaphore>` limits to `Config::MAX_CONCURRENT_DOWNLOADS` (10)
concurrent sessions. Excess requests receive HTTP `429 Too Many Requests`.

#### [Android] `MODE_READ_WRITE` on download file descriptor
**Before:** Downloads opened the destination file with `MODE_READ_WRITE`, which
is wider than necessary and increases attack surface.  
**After:** `MODE_WRITE_ONLY | MODE_CREATE` on Android; `O_WRONLY | O_CREAT` on iOS.

#### [Rust] Patch integrity not verified
**Before:** `apply_patch_fds` applied the bsdiff patch with no integrity checks.
A corrupted patch would silently produce a corrupt output.  
**After:**
  - SHA-256 of the **patch file** is verified before `bspatch` runs.
  - SHA-256 of the **output file** is verified after `bspatch` completes.
  Both checks use the values from the signed manifest.

#### [Rust / Python] Patch format mismatch risk removed
**Before:** `forge.py` produced `bsdiff4` patches while Rust used a different
patching backend and lacked a cross-tool compatibility proof.  
**After:** Rust patching uses a BSDIFF40-compatible backend and includes a
fixture test generated by Python `bsdiff4`.

#### [Rust / Android] Resumed download wrote from the wrong offset
**Before:** resume requests asked the seeder for `Range: bytes=N-`, but the
destination fd could still write from byte `0`, corrupting the file.  
**After:** Android seeks the fd before handing it to Rust, and Rust also seeks
defensively when `resume_from > 0`.

#### [Rust] Peer auth lacked per-model authorization
**Before:** any peer with a valid shared secret could request any registered
`model_id`.  
**After:** peers must be explicitly granted access per model via
`grant_peer_model_access`, and unauthorized downloads return `403`.

#### [Android / iOS] Heartbeat could silently die
**Before:** A single exception in the heartbeat coroutine/Task would kill it
without any log, leaving the service apparently alive but with a dead engine.  
**After:** Bounded retry loop with exponential back-off (5 s → 10 s → 20 s → 30 s
cap). After `MAX_HEARTBEAT_FAILURES` consecutive failures the Rust server is
restarted automatically.

#### [iOS] `AetherEngine` initialised on the main thread
**Before:** `AetherManager.init()` constructed `AetherEngine()` synchronously,
spinning up a Tokio runtime on the main run loop.  
**After:** Construction is done inside `Task.detached(priority: .userInitiated)`.

## [2.2.0] – 2026-04-12

Initial release.

- Rust core with UniFFI bridge (Android + iOS).
- Zero-copy TCP download via file descriptors.
- bsdiff surgical patching.
- HMAC-SHA256 ticket authentication with anti-replay.
- `SecureVault` (Android Keystore TEE) / `Vault` (iOS Secure Enclave).
- `forge.py` admin tool (keygen, publish, verify).
